Local use: this folder must be served over HTTP (not opened as a raw file) so the browser can load app.py and bumbl_index_utils.py: cd hprcv2_pyodide_browser && python3 -m http.server 8000 then open http://127.0.0.1:8000/

Host on a remote server, use from your laptop

Pyodide runs in your browser; the server only serves static files (like notebook HTML). Copy this whole hprcv2_pyodide_browser directory to the host, then on the host run:

cd hprcv2_pyodide_browser && python3 -m http.server 8000 --bind 0.0.0.0

Option A — SSH port forward (recommended): on the host bind to loopback only: python3 -m http.server 8000 --bind 127.0.0.1 then on your laptop: ssh -N -L 8000:127.0.0.1:8000 you@remote and open http://127.0.0.1:8000/ in a local browser.

Option B — direct URL: if your network allows, open http://<remote-ip>:8000/ (and open firewall / security group for that port). Prefer HTTPS and auth in front of the static files if the page is exposed.
